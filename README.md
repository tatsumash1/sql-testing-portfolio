# SQL Testing Portfolio

Портфолио-проект по SQL-тестированию учебной базы данных Northwind.

Проект показывает базовые навыки QA при работе с базой данных: анализ структуры таблиц, написание SQL-запросов, проверка корректности данных, поиск ошибок в связях между таблицами и оформление результатов тестирования.

## Цель проекта

Показать умение использовать SQL для проверки качества данных и бизнес-логики приложения.

В проекте будут выполнены проверки:

- корректность связей между таблицами;
- наличие заказов без клиента;
- наличие заказов без товаров;
- проверка данных клиентов;
- проверка данных товаров;
- проверка поставщиков и категорий;
- проверка статусов и дат заказов;
- проверка количества товаров в заказах;
- поиск некорректных или подозрительных значений;
- выявление тестовых дефектов в данных.

## Тестируемая база данных

В качестве тестируемой базы используется Northwind SQLite:

[jpwhite3/northwind-SQLite3](https://github.com/jpwhite3/northwind-SQLite3)

Northwind - учебная база данных, которая моделирует работу торговой компании. В ней есть клиенты, заказы, товары, сотрудники, поставщики, категории товаров и доставка.

## Основные таблицы

- `Customers` - клиенты;
- `Orders` - заказы;
- `Order Details` - товары внутри заказа;
- `Products` - товары;
- `Categories` - категории товаров;
- `Suppliers` - поставщики;
- `Employees` - сотрудники;
- `Shippers` - службы доставки.

## Что будет внутри

В проекте будут подготовлены SQL-запросы для проверки качества данных в базе Northwind.

Также будут оформлены тестовая документация, чек-лист, тест-кейсы, баг-репорты и итоговый тест-отчет.

Так же будут решены задачи по Sql для практики SQL-запросов и QA-проверок на SqlAcademy.

## Примеры SQL-проверок

В рамках проекта будут использоваться запросы для:

- выборки данных через `SELECT`;
- фильтрации через `WHERE`;
- сортировки через `ORDER BY`;
- группировки через `GROUP BY`;
- проверки агрегированных значений через `COUNT`, `SUM`, `AVG`;
- объединения таблиц через `JOIN`;
- поиска отсутствующих связей через `LEFT JOIN`;
- проверки бизнес-правил через условия;
- поиска дубликатов и пустых значений;
- проверки корректности дат и числовых значений.

## Примеры QA-сценариев

- проверить, что каждый заказ связан с существующим клиентом;
- проверить, что каждый заказ содержит хотя бы один товар;
- проверить, что товары в заказах существуют в таблице `Products`;
- проверить, что у товаров указаны категории и поставщики;
- найти клиентов без заказов;
- найти товары, которые ни разу не были заказаны;
- проверить, что количество товара в заказе больше нуля;
- проверить, что цена товара не равна нулю;
- проверить, что дата отправки заказа не раньше даты создания заказа.

## Демонстрируемые навыки

- SQL-запросы для QA-задач;
- проверка тестовых данных;
- анализ структуры базы данных;
- поиск дефектов в данных;
- проверка связей между таблицами;
- работа с тестовой документацией;
- оформление результатов тестирования.
